Git og Github

En revolusjonerende Idé i å utarbeide digitalt samarbeid

Git?

Sist endret: 01.07 (03.10.17)

- "Git" er en data-funksjon som lar flere brukere jobbe på samme oppgaver samtidig og blir som oftes brukt til koding. "Github" er en av mange nettsider der brukere kan hjelpe/veilede hverandre for å bli en bedre kodere.


Litt historie

"Git" ble først introdusert i 2005 av "Linus Torvalds" i et prosjekt for "Linux Kernel",

sammen med nåværende velikeholder junio Hamano.

Systemet basererte seg på koding-redigering, loggføring og kordinasjon av data filer,

der brukere kunne velikeholde data samtidig.

Navnet fikk den av da Linus skrev den første versjonen og beskrivde den som den: "The stupid content tracker".

3.april(2005) startet prosjektet og det tok ikke lang tid før prosjektet ble annonsert den 6. april samme året.

Den første kjeden av tilkoblinger ble opprettet den 18.april.

Github

Git er idag en gratis programvare som er distribuert under lisensen "GNU General Public Licence",

som tilsier idag at mange levrandører har rettigheten til å ta i bruk programvaren for egen virksomhet.

GitHub er en web-basert versjon av "git" der brukere har mulighet til å motta kontrollerende-samarbeid/ fjernstyrt,

bug-feilmeldinger, tilbakemeldinger og forespørsler.

Hensikten bak konseptet basererer seg på å vidreformidle kunnskap innenfor koding videre.

Fordeler?

For studenter er github et flott verktøy for kunnskap og forstårelse av koding.

Git gir derfor gratis pakker ut til studenter, kalt "Student Developer package".

Her kan studenter få tilgang til alt innenfor programmering gratis eller til en billig penge.

GitHub er en super måte å få tilbakemeldinger,

på hva som er galt eller kunne bli gjort bedre.
